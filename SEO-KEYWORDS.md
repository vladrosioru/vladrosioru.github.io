# SEO Keyword Strategy — vladrosioru.github.io

> **How this works:** Search engines do NOT read this file, and they ignore the
> `<meta name="keywords">` tag. Rankings come from these phrases appearing
> **naturally in the visible page content** — `<title>`, headings, body text,
> image `alt` text, and the JSON-LD structured data. This file is a planning
> reference: it lists each target phrase, how realistic it is, and where on the
> page it is reinforced. When you want to rank for a new phrase, add it here,
> then make sure it actually appears in the visible copy.

## Goal
Be found by people searching for an **engineering manager / leader / coordinator
of software development projects or departments** — with emphasis on the
long-tail variants (role + location + tech + domain), which are winnable, rather
than the bare generic terms, which are dominated by LinkedIn/Indeed/big firms.

## Tier 3 — Aspirational only (too competitive for a new personal site)
Do not expect traffic from bare terms; keep them only for context:
`engineering manager`, `software development manager`, `engineering leader`,
`team lead`, `scrum master`, `agile`, `.NET`, `C#`, `Azure`, `AWS`, `fintech`.

## Rules to keep it effective
1. **One primary phrase per page region.** Don't repeat the same phrase 10×
   (keyword stuffing → Google penalty). Aim for natural reading.
2. **New target phrase → add a row above AND edit the visible copy** so the words
   actually appear together in a sentence, heading, or alt text.
3. **Title + H1 carry the most weight.** Keep your name + top role + location there.
4. **Inbound links matter more than any on-page tweak.** Link this site from
   LinkedIn, GitHub, and any bio/directory. A new site with no links ranks low
   no matter how good the keywords are.
5. **Re-check rankings** by searching the phrase in an incognito window after the
   site is indexed (give Google 1–3 weeks post-submission).
